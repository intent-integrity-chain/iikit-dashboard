# Research: Constitution Radar Chart

## Decisions

### 1. Inline SVG over charting library
**Decision**: Draw radar chart with inline SVG and basic trigonometry.
**Rationale**: A radar/spider chart is a regular polygon with axis lines. The math is: `x = cx + r * cos(angle)`, `y = cy + r * sin(angle)` for each principle. No library needed for this. Keeps dependencies at zero (Constitution V: Simplicity).
**Alternatives considered**: D3.js (rejected — massive dependency for one chart), Chart.js (rejected — adds build complexity), Canvas API (rejected — SVG is more accessible and stylable with CSS).

### 2. CONSTITUTION.md parsing strategy
**Decision**: Parse by markdown heading structure. Principles are `### N. Name` headings under `## Core Principles`. Obligation level is the strongest MUST/SHOULD/MAY keyword found in the principle text. Rationale is text after `**Rationale**:`.
**Rationale**: The constitution template uses a consistent structure. Heading-based parsing is resilient to formatting variations within principle text.
**Alternatives considered**: Regex-only parsing (rejected — fragile across formatting), frontmatter/YAML (rejected — constitutions are pure markdown).

### 3. Version metadata extraction
**Decision**: Parse the footer line pattern `**Version**: X.Y.Z | **Ratified**: DATE | **Last Amended**: DATE`.
**Rationale**: All constitutions generated by `/iikit-00-constitution` use this footer format. Simple regex match.
